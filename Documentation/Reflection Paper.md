Reflection Paper – Dormitory and Cafeteria Management System
Authors: Flutura and Tringa Hyseni 
Project: Dormitory and Cafeteria Management System (C# Console Application & SQL Database)

1. Learning Outcomes
My Reflections
What I learned:
I learned how to design and implement a multi-layered console application in C#, including entities, services, repositories, and connecting to a relational SQL database. I strengthened my understanding of database relationships, primary/foreign keys, and normalization principles. Debugging complex SQL queries and handling exceptions in C# improved my problem-solving skills.

Tringa's Reflections
What I learned:
I learned to work with SQL databases efficiently, including creating tables, inserting data, and constructing queries that interact with a console application. I improved my ability to design database schemas and understand how application logic should correspond to the database structure.

Shared Learnings

Collaboration: Working together helped us merge knowledge of application logic (C#) with database design (SQL). We learned to communicate effectively, share debugging strategies, and align project architecture across both domains.
AI Usage: We learned how to use AI as a tool for explanation, debugging, and best practice guidance, without relying on it for actual code writing or design decisions.


2. Biggest Challenge and How We Solved It
My Challenge
Challenge: Ensuring the C# console application correctly mirrored database updates and reflected real-time data.
Solution: I focused on using proper database connection management, executed queries in a structured way, and validated results by testing each function thoroughly before integrating features.

Tringa's Challenge
Challenge: Handling database errors, missing tables, and ensuring consistency in table relations.
Solution: I systematically checked table structures, validated SQL syntax, and coordinated with Flutura to align entity names and relationships, ensuring the application could interact correctly with the database.

Shared Challenge
Challenge: Integrating the C# logic with the SQL database in a cohesive manner.
Solution: We divided responsibilities: I focused on application logic and Tringa on database integrity. We collaborated on the integration points and performed joint testing to ensure smooth communication between the two layers.

3. How AI Helped/Hindered

AI Assistance
AI explained complex concepts and debugging steps, helping us understand error messages and best practices.
It provided guidance on design principles, database relationships, and code structure without generating code for us.
AI Limitations
AI could not make project-specific architecture decisions for us.
It sometimes suggested generic solutions, requiring us to adapt them to our project context.
We had to manually validate and implement all AI suggestions, ensuring we fully understood and controlled the logic.

4. What We Would Do Differently
I
I would start with a clear diagram connecting C# classes with SQL tables to visualize relationships early.
I would also write unit tests for critical functions earlier in the development cycle to reduce debugging time.

Tringa
I would perform more proactive validation of database constraints and relationships before writing queries.
I would create a small prototype for database interactions before integrating them into the full application.

Shared Plan
Allocate time for integrated testing early, so application logic and database interactions can be validated simultaneously.
Maintain a joint log of AI interactions to document learning points and avoid redundancy.
Improve collaboration workflow with version control branches to prevent conflicts during integration.

Conclusion:
This project was an enriching collaborative experience. I strengthened application logic and C# skills, while Tringe reinforced database design and SQL skills. Together, we learned to merge our expertise, use AI responsibly, and reflect on our development practices to improve future projects.
